# Aprendizados do Projeto — Radar Fundamentalista

## Visão Geral

O projeto Radar Fundamentalista foi desenvolvido como uma experiência prática para conectar conceitos de Produto, Dados, SQL, PostgreSQL, Linux e documentação técnica.

A proposta foi criar um projeto de portfólio com foco em análise de empresas brasileiras por meio de indicadores fundamentalistas, estruturando desde a visão do produto até a criação de banco de dados e consultas SQL.

---

## Principais Aprendizados

### 1. Organização de Projeto Técnico

Aprendi a estruturar um projeto em pastas, separando documentação, dados, scripts SQL e arquivos de dashboard.

Estrutura criada:

* `docs/` para documentação;
* `data/` para bases de dados;
* `scripts/` para arquivos SQL;
* `dashboard/` para futuras visualizações.

Essa organização facilita a manutenção, leitura e apresentação do projeto.

---

### 2. Uso do Terminal Linux

Durante o projeto, pratiquei comandos básicos de navegação e manipulação de arquivos no Linux.

Comandos utilizados:

* `cd` para entrar em pastas;
* `cd ..` para voltar uma pasta;
* `ls` para listar arquivos;
* `pwd` para verificar o caminho atual;
* `mkdir` para criar pastas;
* `nano` para criar e editar arquivos;
* `cat` para ler e executar arquivos;
* `find` para localizar arquivos.

Esse aprendizado foi importante para ganhar autonomia no ambiente Linux.

---

### 3. Documentação Técnica em Markdown

Aprendi a criar arquivos `.md` para documentar o projeto de forma organizada.

Documentos criados:

* README do projeto;
* Visão do produto;
* Backlog;
* Métricas;
* Roadmap;
* Persona e hipóteses;
* MVP;
* Critérios de classificação;
* Dicionário de dados;
* Observações sobre a base;
* Comandos PostgreSQL.

Essa documentação ajuda a explicar não apenas o que foi feito, mas também o raciocínio por trás das decisões.

---

### 4. Modelagem de Dados

Aprendi a pensar na estrutura de uma base de dados, definindo campos, tipos de informação e finalidade de cada coluna.

A base foi estruturada com informações como:

* ticker;
* empresa;
* setor;
* preço;
* P/L;
* ROE;
* ROIC;
* margem líquida;
* dividend yield;
* dívida líquida/EBITDA;
* liquidez corrente;
* classificação.

Também foi criado um dicionário de dados para explicar o significado de cada campo.

---

### 5. PostgreSQL e SQL

Aprendi a criar um banco de dados no PostgreSQL e executar comandos SQL para estruturar e consultar informações.

Foram praticados conceitos como:

* criação de banco de dados;
* criação de tabela;
* inserção de dados;
* consultas com `SELECT`;
* ordenação com `ORDER BY`;
* agrupamento com `GROUP BY`;
* contagem com `COUNT`;
* filtros com `WHERE`.

Esse aprendizado conectou teoria de dados com aplicação prática.

---

### 6. Produto e Dados

O projeto reforçou a importância de conectar dados com uma necessidade real do usuário.

Antes de criar a base e as consultas, foram definidos:

* problema;
* público-alvo;
* persona;
* hipótese;
* proposta de valor;
* MVP;
* roadmap;
* métricas;
* critérios de sucesso.

Isso ajudou a transformar o projeto em um case de Produto, e não apenas em um exercício técnico.
